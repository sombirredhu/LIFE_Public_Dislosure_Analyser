## 1. LLM Provider тАФ `src/llm_client.py`

### What It Is
Uses the **Anthropic SDK** directly (`anthropic==0.25.0`). Single model, no cost tiers.

```python

from src.config import ANTHROPIC_API_KEY, CLAUDE_MODEL, CLAUDE_MAX_TOKENS, CLAUDE_TEMPERATURE

def ask_claude(system_prompt, user_message, max_tokens=None, temperature=None, max_retries=3) -> str:
    client = Anthropic(api_key=ANTHROPIC_API_KEY)
    response = client.messages.create(model=CLAUDE_MODEL, ...)
    return response.content[0].text
```

### What It Should Be (per `07_modules.md` ┬з 7.6)
Uses the **OpenAI SDK** pointed at **OpenRouter**. Two-tier routing тАФ free model for simple queries, paid model for complex.

```python

from src.config import OPENROUTER_API_KEY, OPENROUTER_BASE_URL, LLM_MODEL_FREE, LLM_MODEL_PAID, \
                       LLM_MAX_TOKENS_SIMPLE, LLM_MAX_TOKENS_COMPLEX, LLM_TEMPERATURE

def ask_llm(system_prompt: str, user_message: str, use_paid: bool = False) -> str:
    """
    use_paid=False тЖТ model=LLM_MODEL_FREE,  max_tokens=LLM_MAX_TOKENS_SIMPLE (1024)
    use_paid=True  тЖТ model=LLM_MODEL_PAID,  max_tokens=LLM_MAX_TOKENS_COMPLEX (4096)
    """
    client = OpenAI(base_url=OPENROUTER_BASE_URL, api_key=OPENROUTER_API_KEY)
    # Retries: 3 attempts with exponential backoff
    # Timeout: 30 seconds per request
    # Must check finish_reason == 'length' тЖТ log warning if truncated
    # Must log model + tokens used on every call (for cost tracking)
```

## 3. `src/rag_pipeline.py` тАФ Missing Core Logic

### What It Is
Single-path pipeline: retrieve тЖТ build prompt тЖТ call Claude. No complexity detection, no top-up, no token guard, no model logging.

```python

user_message = build_user_message(question, chunks)
confidence = get_confidence_level(chunks)
answer = ask_claude(SYSTEM_PROMPT, user_message)

        "retrieved_chunks": chunks}   # extra field not in plan
```

### What It Should Be (per `07_modules.md` ┬з 7.7)

```python
# Required answer_question() flow:
complexity = classify_complexity(question)           # pure keyword heuristic, no LLM call

if complexity == "simple":
    chunks = retrieve(question, filters, top_k=TOP_K_SIMPLE)
else:  # complex
    chunks = retrieve(question, filters, top_k=TOP_K_COMPLEX)
    indexed = embedder.get_indexed_companies()       # who is in ChromaDB?
    chunks = top_up_missing_companies(question, chunks, indexed, filters)

# Input token budget guard (MUST run before LLM call):
total_chars = sum(len(c['text']) for c in chunks)
if total_chars > LLM_MAX_INPUT_CHARS:
    chunks = chunks[:LLM_MAX_INPUT_CHARS // avg_chunk_size]
    log warning: f"Input truncated: {total_chars} chars exceeded {LLM_MAX_INPUT_CHARS} limit"

answer = ask_llm(SYSTEM_PROMPT, user_message, use_paid=(complexity == "complex"))

# Return dict MUST include model_used:
return {
    "answer":      str,
    "sources":     list,
    "chunks_used": int,
    "confidence":  "high" | "medium" | "none",
    "model_used":  str    # e.g. "anthropic/claude-3-haiku:free" тАФ for cost visibility
}
# Note: "retrieved_chunks" should NOT be in the return dict per plan
```

### Missing: `classify_complexity()` function

#   "industry total", "channel-wise", "rank", "ranking", "which company"

#   "highest", "lowest", "top", "bottom", "best", "worst", "most", "least", "trend"
#

## 4. `src/retriever.py` тАФ Missing `top_up_missing_companies()`

### What It Is
Has `retrieve()` plus extra helper functions (`retrieve_by_company`, `retrieve_by_period`, `retrieve_tables_only`) not specified in the plan. The key function `top_up_missing_companies()` is entirely absent.

Also: `get_confidence_level()` returns `"low"` as a possible value тАФ the plan explicitly prohibits this.

Score conversion formula is wrong:
- **Actual:** `similarity = 1 / (1 + distance)` (L2 distance formula)
- **Plan:** `score = 1 - distance` (cosine distance formula тАФ ChromaDB with cosine metric)

### What It Should Be (per `07_modules.md` ┬з 7.5)

```python
def top_up_missing_companies(
    query: str,
    chunks: list,
    expected_companies: list,
    filters: dict
) -> list:
    """
    1. Find which companies from expected_companies are missing in retrieved chunks
    2. Fetch missing companies IN PARALLEL using ThreadPoolExecutor(max_workers=8):
       with ThreadPoolExecutor(max_workers=8) as pool:
           futures = {pool.submit(retrieve, query, {"company_code": co}, 2): co
                      for co in missing_companies}
    3. Merge top-up results into main chunk list
    """
```

`get_confidence_level()` must return only `"high"` / `"medium"` / `"none"` тАФ never `"low"`:

```python
# Required thresholds (SIMILARITY_THRESHOLD=0.4 pre-filters, so 'low' never reaches here):
if top_score >= 0.7:   return "high"
if top_score >= 0.4:   return "medium"
return "none"
# "low" is NOT a valid return value per 08_system_prompt.md
```

## 7. `src/chunker.py` тАФ Structural and Content-Type Gaps

### What It Is

Chunk dict has **3 top-level keys**: `chunk_id`, `text`, `metadata`.
No `summary` or `header` content types тАФ only `table` and `text`.
Section detection is keyword-based in chunker (not from L-page index in pdf_parser).

```python
# Current chunk structure:
{
    "chunk_id": "HDFC_Life_Q1_FY25_page3_chunk2",   # тЖР top-level key
    "text": "...",
    "metadata": { ... }
}
```

### What It Should Be (per `07_modules.md` ┬з 7.3 and `06_chunk_metadata.md`)

```python
# Required chunk structure тАФ only 2 top-level keys:
{
    "text":     "...",
    "metadata": {
        "chunk_id":     "HDFC_Life_Q1_FY25_page3_chunk2",  # inside metadata
        "page_label":   "L-5",                              # from pdf_parser page_label
        "section":      "Analytical Ratios",                # from pdf_parser section
        "content_type": "table" | "text" | "summary" | "header",
        ...
    }
}
```

**Missing `content_type` values:**

| Value | Required Detection Rule |
|-------|------------------------|
| `summary` | Section name contains "summary"/"highlights"/"key metrics"/"overview"/"executive" OR page 1 with section="unknown" OR text starts with "Key Highlights"/"Executive Summary"/etc. |
| `header` | Line with no sentence-ending punctuation, under 100 chars |
| `table` | тЬЕ implemented |
| `text` | тЬЕ implemented |
